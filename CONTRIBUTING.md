# Working on this course

1. Read `_meta/master-spec.md` and `_meta/STYLE-GUIDE.md` before touching a module.
2. Edit Markdown in `modules/NN-name/src/` only. Treat `build/` as generated.
3. Every module holds the 4-slot skeleton; the through-line lives in slot 2, by rhyme.
4. Flag build-sensitive claims; note the target TD build (especially Modules 0 and 4).
5. Update `_meta/STATUS.md` at the end of a session.

## Per-chat continuity (no shared live state)

This repo + the master spec ARE the memory. To continue work in a new chat, attach
`_meta/master-spec.md` and the relevant module's `src/` files, plus `STATUS.md`.
